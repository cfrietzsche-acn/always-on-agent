# Sprint Issues

Issue docs for the always-on-agent build. Each issue is created by PM, reviewed by SDM, built by Dev, and signed off by QA before the associated commit is pushed to GitHub.

## Naming convention

`issue-{kebab-description}-{XX}.md`

Examples: `issue-tools-01.md`, `issue-agent-loop-02.md`, `issue-triage-prompt-03.md`

## Status lifecycle

`Draft` → `SDM Review` → `In Progress` → `QA` → `Done`
